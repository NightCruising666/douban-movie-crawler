# 数据清洗方案与实际结果

## 1. 清洗目标

清洗过程遵循三项原则：

1. `data/movies.csv` 是阶段二原始事实表，任何清洗都不覆盖它。
2. 清洗主表尽量保留记录，通过质量标记和课题纳入标志区分用途。
3. 每条规则同时记录判定标准、处理方式、选择原因和未采用方法，便于实验报告复现。

运行命令：

```bash
python douban_crawler/run_cleaning.py
```

输出位于 `douban_crawler/data/processed/`：

- `movies_cleaned.csv`：清洗后的电影主表；
- `movies_cleaning_report.csv`：汇总质量指标；
- `movies_cleaning_rules.csv`：逐条规则及命中数量；
- 后续存在正式 `reviews.csv` 时，额外生成短评清洗表和短评指标表。

## 2. 电影表清洗规则

| 规则 | 处理对象 | 判定与处理 | 为什么这样做 | 未采用方法 |
|---|---|---|---|---|
| R01 | 文本字段 | 去除多余空白和占位符，统一多值分隔符 | 统一格式，不改变实体含义 | 不对片名、人名做模糊合并 |
| R02 | 豆瓣ID重复 | 按采集时间保留最新快照 | 一个实体保留一个分析时点 | 不按电影名称去重，同名可能是不同作品 |
| R03 | 豆瓣ID缺失 | 清洗表删除，原始表保留 | 无法可靠关联标签和短评 | 不人工生成替代ID |
| R04 | 评分 | 评分须在 `(0,10]` 且评价人数大于0，否则置为空 | 豆瓣返回的0表示未形成评分，不等于作品得0分 | 不填0，也不用均值填补 |
| R05 | 计数字段 | 评价人数、短评数、长评数须为非负整数 | 防止错误数值进入比率和权重 | 不四舍五入，不裁剪为0 |
| R06 | 上映年份 | 1888年至原始时区采集年份后1年；单条采集时间无效时使用数据集最大有效采集年；若整列均无效则使用研究结束年+1作为采集年 | 兼顾电影史边界和待上映记录，避免UTC跨年误判，且二级回退可审计 | 不直接删除整行 |
| R07 | 首映日期 | 区分日、月、年、缺失和异常精度 | 保留API实际精度 | 不把只有年份的数据虚构为1月1日 |
| R08 | 片长 | 从原始文本提取首个1—1440分钟整数 | 原文可能包含地区或版本说明 | 不覆盖原始片长文本 |
| R09 | 产地 | 中国大陆、合拍、港澳台、进口、未知五类 | 合拍片不宜强行二分 | 不只分国产/进口 |
| R10 | 课题A样本 | 评分大于0且评价人数大于0 | 评分稳定性分析需要有效评分及分母 | 不因评价人数少而删除小众作品 |
| R11 | 课题B样本 | 上映年份为2005—2025 | 与研究时间边界一致 | 不从清洗主表提前删除其他年份 |
| R12 | 贝叶斯评分 | 总体均值为 `C`，评价人数中位数为 `m` | 降低小样本极端评分的波动 | 不设置单一评价人数硬门槛 |
| R13 | 采集时间 | 缺失、不能解析或原文不含时区 | 保证去重时“最新快照”有可靠依据 | 不把无时区时间直接当UTC，也不伪造时间 |

| 指标 | 数量 | 处理结果 |
|---|---:|---|
| 原始记录 | 2601 | 原始表不修改 |
| 重复豆瓣ID | 0 | 无记录被去重删除 |

| 有效评分样本 | 2590 | 纳入课题A |
| 无有效评分 | 11 | 评分置为空；记录及短评总数保留 |

## 4. 两个课题的筛选边界

### 课题A：评分统计稳定性

- 使用2590条有效评分记录；
- 不按评价人数设置硬删除门槛；
- 同时保留原评分、评价人数、短评/长评参与率和贝叶斯加权评分；
- 小众作品仍参与分析，通过贝叶斯收缩和敏感性分析表达不确定性。

### 课题B：类型结构变化

- 使用2005—2025年的1828条记录；
- 多类型作品后续按 `1/类型数` 分配权重，确保每部作品总权重为1；
- 研究结论表述为“豆瓣标签样本中的结构变化”，不外推为完整电影市场；
- 2005年前和2025年后的记录不删除，只是不进入该子课题。

## 5. 可复现性

清洗输出属于可再生产物，默认不提交到Git。任何电脑拉取仓库后，安装依赖并运行清洗命令，都应从同一份 `movies.csv` 生成相同的记录数量、纳入标志和质量报告。

## 6. 涉及的知识点

- **主键治理**：使用豆瓣ID识别实体，不能仅靠片名去重。
- **缺失值语义**：0分与缺失评分含义不同；缺失不能随意填成均值。
- **可空数据类型**：计数使用 Pandas `Int64`，允许缺失同时保持整数语义。
- **时间精度**：年、年月、年月日是三种不同信息量，不能强制补齐。
- **多值分类**：类型和国家地区先标准化，再在统计阶段展开。
- **分析样本与清洗主表分离**：清洗不等于删数据，不同课题使用不同纳入标志。
- **经验贝叶斯收缩**：小样本评分向总体均值收缩，用于表达不稳定性。
- **审计追踪**：规则表记录标准、处理、理由、备选方案和实际命中数量。

## 7. 常见错误及解决办法

| 常见错误 | 后果 | 本项目的解决办法 |
|---|---|---|
| 把评分0当成真实0分 | 人为拉低均值 | 评分置为空，并用质量标记保留证据 |
| 对缺失评分填总体均值 | 制造不存在的评分 | 不插补，只从评分分析中排除 |
| 按电影名称去重 | 误删同名不同作品 | 只按豆瓣ID去重 |
| 把只有年份的日期变成1月1日 | 虚构时间精度 | 保留原文并增加日期精度字段 |
| 为了课题B直接删除其他年份 | 课题A损失老片和新片 | 主表全部保留，只使用纳入标志 |
| 多类型作品每类都计1部 | 年度总量被重复放大 | 后续按 `1/类型数` 分配权重 |
| 将合拍片全部算作国产或进口 | 产地比较偏差 | 单独设置“合拍”分类 |
| 清洗脚本覆盖原始CSV | 无法复核和重做 | 只写入 `data/processed/` |

## 8. 如何扩展到最终项目

1. 阶段三短评采集完成后，沿用稳定主键和独立断点，生成 `reviews_cleaned.csv`。
2. 对短评星级比较普通平均、原始有用数加权和对数有用数加权，避免少数超高有用数支配结果。
3. 将清洗报告中的实际数量直接引用到实验报告“数据治理”章节。
4. 运行 `run_analysis.py` 生成课题A、课题B的统计表和图片。
5. 报告中明确说明标签采样偏差、热门短评偏差和快照时间边界。
6. PPT只展示核心规则和异常数量，完整规则表作为附录或仓库证据。

完整的阶段筛选、方案比较和决策记录见 [PIPELINE_DECISIONS.md](PIPELINE_DECISIONS.md)。
