## 快速开始

### 环境要求

- Python 3.9+
- Git

### 1. 克隆仓库

```bash
git clone https://github.com/NightCruising666/douban-movie-crawler.git
cd douban-movie-crawler
```

### 2. 安装依赖

```bash
pip install -r douban_crawler/requirements.txt
```

### 3. 继续采集

```bash
# 阶段二：电影详情（边爬边存，断点续传）
python douban_crawler/run_batch.py
python douban_crawler/run_batch.py --status    # 看进度

# 阶段三：短评采集
python douban_crawler/run_stage3.py
python douban_crawler/run_stage3.py --status    # 看进度
```

> 💡 遇到 IP 被限？切换网络（Wi-Fi ↔ 手机热点）后重跑即可。

## 数据采集流程

```
┌────────────┐      ┌──────────────┐      ┌────────────┐
│  阶段一     │ ───→ │  阶段二       │ ───→ │  阶段三     │
│  搜索API    │      │  Rexxar 详情  │      │  Rexxar 短评 │
│  19 标签    │      │  12 字段      │      │  6 字段      │
│  2478 部    │      │  2478 部全量  │      │  30 条/部    │
└────────────┘      └──────────────┘      └────────────┘
movies_raw.csv       movies.csv             reviews.csv
```

### 阶段一：电影列表采集 ✅

- **API**：`movie.douban.com/j/search_subjects`
- **标签**：热门、最新、经典、可播放、豆瓣高分、冷门佳片、华语、欧美、韩国、日本、动作、喜剧、爱情、科幻、悬疑、恐怖、剧情、动画、纪录片（19 个）
- **去重**：按豆瓣 ID 去重
- **产出**：2478 部 → `movies_raw.csv`

### 阶段二：电影详情采集 🔄

- **API**：`m.douban.com/rexxar/api/v2/movie/{id}`
- **目标**：2478 部全量
- **字段**（12 个）：电影名称、导演、主演、上映年份、类型、国家/地区、片长、豆瓣评分、评价人数、短评总数、长评总数、五星占比
- **五星占比**：API 不直接返回，在数据清洗阶段从短评星级分布中计算

### 阶段三：短评采集 ⬜

- **API**：`m.douban.com/rexxar/api/v2/movie/{id}/interests`
- **目标**：2478 部 × 30 条 ≈ 7.4 万条
- **字段**（6 个）：电影名称、用户名称、评分（星级）、短评正文、有用数、评论时间

## 反爬策略

| 策略 | 说明 |
|------|------|
| **User-Agent** | 模拟 Chrome 浏览器 |
| **随机延迟** | 每次请求间隔 3.5~6.5 秒随机抖动 |
| **主动冷却** | 每 3 部休息 45 秒 |
| **失败容错** | 1-2 次失败等 10s 重试，≥6 次停止 |
| **IP 切换** | 限制后切换 Wi-Fi ↔ 手机热点 |
| **断点续传** | 中断后重跑自动跳过已有数据 |

## 统计分析方案

### 子课题 A：评分可信度

| # | 分析 | 方法 | 预期结论 |
|---|------|------|----------|
| A1 | 评分分布 | 直方图 + KDE | 豆瓣评分集中在 6-8.5 分 |
| A2 | 评价人数 vs 评分 | log(人数) × 评分 散点图 | 人数越多评分越收敛 |
| A3 | 四象限分类 | 高低分 × 多寡评价 | 小众高分集中在文艺片/纪录片 |
| A4 | 加权评分 | Σ(评分×有用数)/Σ(有用数) | 对比豆瓣评分，检验虚高 |
| A5 | 短评覆盖率 | 短评数/评价人数 vs 评分 | 覆盖率低的高分需警惕 |
| A6 | 星级分布一致性 | 每部电影 1-5 星熵值 | 低熵 = 可能控评 |

### 子课题 B：类型流行趋势

| # | 分析 | 方法 | 预期结论 |
|---|------|------|----------|
| B1 | 年度类型占比 | 堆叠面积图（2005-2025） | 科幻/动作上升，爱情/剧情下降 |
| B2 | 国产 vs 进口 | 分组箱线图 | 进口片评分偏高但评价少 |
| B3 | 类型-评分交叉 | 热力图 | 纪录片高、恐怖低 |
| B4 | 国家-类型交叉 | 交叉表 | 美国=科幻/动作，日本=动画 |

## 注意事项

1. **IP 限制**：豆瓣 Rexxar API 约 20-50 次/IP，切换网络即可绕过
2. **数据编码**：CSV 使用 UTF-8 BOM，Excel 可直接打开
3. **断点文件**：`.checkpoint_stage1_tags` 不提交到 Git
4. **旧版数据**：`movies_old.csv` / `reviews_old.csv` 为第一轮采集备份，可安全删除
