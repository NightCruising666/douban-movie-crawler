**豆瓣电影评分可信度与类型流行趋势研究**

- **子课题 A**：电影评分可信度分析 — 评分分布特征、评价人数与评分关联性、高分 vs 小众差异

## 快速开始

### 环境要求

- Python 3.9+
- Git

### 1. 克隆仓库

```bash
git clone https://github.com/NightCruising666/douban-movie-crawler.git
cd douban-movie-crawler
```

### 2. 安装依赖

```bash
pip install -r douban_crawler/requirements.txt
```

### 3. 查看当前进度

```bash
python douban_crawler/run_batch.py --status
```

### 4. 继续采集

```bash
python douban_crawler/run_batch.py
```

> 💡 遇到 IP 被限？切换网络（Wi-Fi ↔ 手机热点）后重跑即可。
> 每个 IP 约可采集 20-50 部电影。

## 数据采集流程

```
┌──────────┐      ┌──────────┐      ┌──────────┐
│ 阶段一    │ ───→ │ 阶段一→二 │ ───→ │ 阶段二    │
│ 搜索API   │      │ 筛选     │      │ Rexxar API│
│ 19标签    │      │ 500部    │      │ 10字段    │
│ 2478部    │      │          │      │ 边爬边存  │
└──────────┘      └──────────┘      └──────────┘
movies_raw.csv                      movies.csv
（完整保留）                          （详情数据）
```

### 阶段一：电影列表采集

- **API**：`movie.douban.com/j/search_subjects`
- **标签**：热门、最新、经典、可播放、豆瓣高分、冷门佳片、华语、欧美、韩国、日本、动作、喜剧、爱情、科幻、悬疑、恐怖、剧情、动画、纪录片（19 个）
- **去重**：按豆瓣 ID 去重
- **断点**：标签级别，每个标签完成后写入 `.checkpoint_stage1_tags`

### 阶段一→二：筛选

- **方法**：保留前 500 部（按搜索 API 返回顺序）
- **依据**：豆瓣搜索 API 在每个标签内按相关性排序；标签顺序覆盖了热度 + 时效性 + 地区 + 类型等多个维度
- **数据保留**：movies_raw.csv 保留全部 2478 部，movies.csv 存 500 部详情

### 阶段二：电影详情采集

- **API**：`m.douban.com/rexxar/api/v2/movie/{id}`
- **字段**：电影名称、导演、主演、上映年份、类型、国家/地区、片长、豆瓣评分、评价人数、五星占比
- **保存**：每部电影即时追加写入 CSV（边爬边存）
- **断点**：电影级别，按电影名称跳过已采集

### 阶段三：短评采集（待开发）

- **API**：`m.douban.com/rexxar/api/v2/movie/{id}/interests`
- **字段**：电影名称、用户名称、评分、短评正文、有用数、评论时间

## 反爬策略

| 策略 | 说明 |
|------|------|
| **User-Agent** | 模拟 Chrome 浏览器 |
| **随机延迟** | 每次请求间隔 3.5~6.5 秒随机抖动 |
| **主动冷却** | 每 15 部休息 45 秒 |
| **失败容错** | 1-2 次失败等 10s 重试，3-5 次等 45s，≥6 次停止 |
| **IP 切换** | 限制后切换 Wi-Fi ↔ 手机热点 |
| **断点续传** | 中断后重跑自动跳过已有数据 |

## 注意事项

1. **IP 限制**：豆瓣对 Rexxar API 限制约 20-50 次请求/IP，切换网络即可绕过
2. **数据编码**：CSV 使用 UTF-8 BOM，Excel 可直接打开
3. **断点文件**：`.checkpoint_stage1_tags` 和 `data/.checkpoint_*` 不提交到 Git
4. **校园网建议**：Wi-Fi 和手机热点交替使用，每轮可采集 20-50 部
